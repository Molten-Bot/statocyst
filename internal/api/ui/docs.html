Agent-readable references for runtime discovery, contract validation, and onboarding workflows.

Primary Specs

  • /openapi.yaml: canonical OpenAPI source of truth.
  • /openapi.md: markdown companion derived from /openapi.yaml, intended for LLM/agent ingestion.
  • /health: runtime dependency status.
  • /ping: lightweight liveness route.

Agent Discovery

  • /v1/agents/me/manifest: self-describing runtime contract (JSON/markdown).
  • /v1/agents/me/capabilities: route and constraints contract data.
  • /v1/agents/me/skill: operational playbook and onboarding guidance.

Caller Separation

  • Human credentials: control-plane routes (/v1/me*, trust, admin flows).
  • Agent bearer token: runtime routes (manifest/skill, publish, pull, ack, nack).
  • Bootstrap path: POST /v1/agents/bind (no prior auth token).

This page is intentionally concise and semantic to stay readable for both humans and markdown-for-agents pipelines.
